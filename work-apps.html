Rams Care Data Dashboard

Streamlit · Python · Data Viz Live

A data dashboard that turns our PBIS survey data into something staff can actually use. Teachers and administrators can slice by grade, section, time period, and question to see how students are experiencing school, and where targeted support might help. Built with Streamlit because fast iteration matters more than polish when you're the only developer.

The "Student Insights" view is the one that gets used most, it surfaces individual response patterns that often get buried in aggregate reports.

Unified Arts Website

Web Design · Google Classroom Integration Live

A redesigned website for the Unified Arts team at Robert Adams, art, music, PE, health, and computer science all under one roof. The previous site was hard to navigate and disconnected from the tools teachers and students actually use day-to-day. The rebuild integrates directly with Google Classroom so homework, announcements, and resources flow into one clean interface for families and students.

The design goal was "feels like a real website, not a school website", calm layout, clear typography, and a homepage that gives equal footing to every discipline.

RAMS Heartbeat

React · School Operations In Progress

An all-in-one platform designed to keep Robert Adams running smoothly by giving staff, students, and administrators a single place to find what they need. Instead of bouncing between disconnected tools and spreadsheets, Heartbeat brings everything into one clean interface built around each user's role.

The staff dashboard surfaces schedules, announcements, and day-to-day logistics so teachers can focus on teaching. The student portal gives kids access to their resources, assignments, and school news in a layout they can actually navigate. Admin tools handle the school-wide view, reports, data, and operational details that keep things organized behind the scenes. A shared communication hub ties it all together with notifications, messaging, and a live news feed so nothing falls through the cracks.

Built in React because the interface needs to feel responsive and fast for users who are checking it between classes, not sitting at a desk. Currently in active development.
